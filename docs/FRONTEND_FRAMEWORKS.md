# Frontend-Frameworks: React vs Svelte vs Angular

Bewertung für eine kleine, echtzeitfähige E2EE-Chat‑WebApp.

## Zusammenfassung
- React: Universell, riesiges Ökosystem, gute TS‑Unterstützung; mehr Boilerplate.
- Svelte: Sehr schlank, geringe Bundle‑Größe, schnelle Umsetzung; kleineres Ökosystem.
- Angular: Voll-Framework mit klarer Struktur, DI/Router/Formen eingebaut; höheres Grundgewicht.

## Kriterien
- Lernkurve: Svelte < React < Angular
- Bundle/Runtime‑Overhead: Svelte (sehr klein) < React < Angular
- Ökosystem/Libs: React (sehr groß) > Angular > Svelte
- Struktur/Guidance: Angular (stark) > React (mittel, via Meta-Framework) > Svelte (leichtgew.)
- DX/TS: React/Svelte sehr gut; Angular strikt und robust
- Eignung MVP: Svelte/React ideal; Angular, wenn Team bereits Angular nutzt

## Empfehlung
- MVP: Svelte (schnell, klein) oder React (breite Lib‑Auswahl). Wenn Team Angular‑Erfahrung hat oder Enterprise‑Strukturen wünscht, Angular.
